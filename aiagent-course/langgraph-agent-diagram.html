第二层：LangGraph 图结构

State（状态）

整个流程共享的"内存"。每个节点读取 State、写回 State，信息在节点间流动。

Node（节点）

一个 Python 函数。接收 State → 执行逻辑（调 LLM / 工具 / 数据库）→ 返回更新后的 State。

Edge（边）

普通边 = 固定走向。
条件边 = 根据 State 动态选择下一个节点，实现分支和循环。

Checkpointer

自动保存每步状态。Agent 可以暂停、回滚、人工介入后继续执行。

StateGraph — 客户需求处理 Agent
第三层：核心代码结构
business_agent.py — LangGraph 骨架
# ① 安装依赖
# pip install langgraph langchain-openai langsmith

from langgraph.graph import StateGraph, END
from typing import TypedDict, Annotated
from langchain_openai import ChatOpenAI

# ② 定义 State ── 全局共享的"记忆"
class AgentState(TypedDict):
    user_input:  str
    intent:      str          # 意图分类结果
    tool_result: str          # 工具返回值
    response:    str          # 最终回复
    quality_ok:  bool         # 质量检查是否通过

# ③ 定义节点函数
llm = ChatOpenAI(model="gpt-4o")

def classify_intent(state: AgentState):
    prompt = f"将需求分类为 search/generate/escalate：{state['user_input']}"
    result = llm.invoke(prompt)
    return {"intent": result.content.strip()}

def search_tools(state: AgentState):
    # 调用搜索工具 / 数据库查询
    return {"tool_result": "[搜索结果]"}

def generate_response(state: AgentState):
    prompt = f"根据 {state['tool_result']} 生成回复"
    result = llm.invoke(prompt)
    return {"response": result.content}

def quality_check(state: AgentState):
    ok = len(state["response"]) > 10  # 简单示例
    return {"quality_ok": ok}

# ④ 条件边函数 ── 路由逻辑
def route_intent(state: AgentState) -> str:
    return state["intent"]  # "search" | "generate" | "escalate"

def route_quality(state: AgentState) -> str:
    return "output" if state["quality_ok"] else "generate"

# ⑤ 构建图
graph = StateGraph(AgentState)

# 添加节点
graph.add_node("classify",  classify_intent)
graph.add_node("search",    search_tools)
graph.add_node("generate",  generate_response)
graph.add_node("escalate",  lambda s: {"response": "已转人工处理"})
graph.add_node("quality",   quality_check)
graph.add_node("output",    lambda s: s)

# 连接边
graph.set_entry_point("classify")
graph.add_conditional_edges("classify", route_intent,
    {"search": "search", "generate": "generate", "escalate": "escalate"})
graph.add_edge("search",   "generate")
graph.add_edge("generate", "quality")
graph.add_edge("escalate", "output")
graph.add_conditional_edges("quality", route_quality,
    {"output": "output", "generate": "generate"})  # retry loop
graph.add_edge("output", END)

# ⑥ 编译 & 运行
app = graph.compile()
result = app.invoke({"user_input": "查一下上周的销售数据"})
🔁
循环 = 真正的智能
quality_check → generate 的回路让 Agent 自动重试，直到结果达标。这是 LangGraph 比线性 Chain 强大的核心。
👤
Human-in-the-Loop
在任何节点加 interrupt_before=["escalate"] 即可让 Agent 暂停，等人工审批后继续执行。
🔭
LangSmith 可观测
设置 LANGSMITH_API_KEY 后，每次运行的节点路径、LLM 调用、耗时全部可视化追踪。
