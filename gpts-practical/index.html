这节课解决什么问题

不从抽象概念开始，先把一个智能体做出来。

很多人听过 AI Agent，却卡在第一步：不知道角色怎么写、资料怎么准备、边界怎么设置、怎样判断它真的可用。 这节课用一个足够小但完整的案例，把这些关键动作走一遍。

课堂成品

你会亲手搭建一个「AI Agent 课程助教」

它不是泛泛聊天机器人，而是一个有岗位说明、有资料来源、有测试标准的学习助理。

你的 v1 成品

AI Agent 课程助教

  • 解释 AI Agent 和普通 ChatGPT 的区别

角色

它是谁，服务谁，以什么语气回答。

任务

它主要帮助用户完成哪些学习动作。

知识

它依据课程大纲、FAQ、术语表回答。

边界

它不会编造资料，也不处理敏感决策。

适合人群

适合想先做出第一个 AI 助手的人

这不是底层模型训练课，也不是企业级自动化部署课。它适合需要一个清晰入口的人。

适合你，如果你

  • 不会编程，但愿意跟着步骤操作
  • 已经会用 ChatGPT，却不知道怎样变成具体助理
  • 想为课程、内容、客服或学习场景做 AI 助手
  • 希望先做无代码原型，再学 Dify、OpenClaw 或 API

暂时不适合，如果你

课程大纲

从想法到可测试 GPT，完整走一遍

  1. 01

    明确今天要做出的成品

    理解课程助教不是普通聊天机器人，而是一个被安排到具体岗位上的 AI 助手。

    5 分钟
  2. 02

    理解 AI Agent 的最小结构

    用零基础语言拆解角色、任务、知识、边界四个组成部分。

    10 分钟
  3. 03

    准备 Knowledge 文件

    区分 Knowledge 和 Instructions，整理课程大纲、FAQ、术语表三类资料。

    10-15 分钟
  4. 04

    在 ChatGPT GPTs 中创建助教

    填写名称、简介、Instructions、Knowledge 和 Conversation starters，先保存为私有版本。

    20-25 分钟
  5. 05

    用测试问题调教它

    检查知识命中、概念解释、学习路径、边界拒绝和资料不足时的诚实度。

    15-20 分钟

学习重点

先学会判断一个 Agent 是否稳定，再谈更复杂的自动化。

第一节课不急着接 API、自动发消息或操作后台。零基础阶段最重要的是看清楚智能体的基本结构： 它为什么这样回答、依据什么资料回答、什么时候应该停下来提醒用户。

当这个 v1 稳定之后，再继续学习 Dify、OpenClaw、API 或工作流自动化，才不会把问题全部堆到工具配置和报错里。

常见问题

课前你可能会问

我完全不会编程，可以学吗？

可以。这节课第一版使用 ChatGPT GPTs，重点是无代码搭建。你需要会打开网页、复制粘贴文字、上传文件，并愿意把需求写清楚。

这节课为什么不一开始就接 API？

因为零基础学员最先需要掌握的是思路：角色怎么定义、资料怎么准备、边界怎么写、怎么测试。API 和自动化是后面的扩展。

这个课程助教能替我操作后台吗？

本课 v1 不做自动操作后台，也不处理账号密码等敏感信息。它先完成课程问答、概念解释和学习路径建议。

学完后能用到别的业务吗？

可以。你可以把课程助教换成客服问答助理、内容选题助理、产品上架助理或英语学习助理，结构仍然是角色、任务、知识、边界和测试。
